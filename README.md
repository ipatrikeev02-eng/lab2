### 3. Алгоритм

#### Алгоритм выполнения программы:

Задача 1.

1. **Ввод данных**  

   Программа считывает 2 целых числа:
   
   - `a0` — начальное значение последовательности;
   
   - `n` — количество элементов, которые необходимо вывести

3. **Вычисление и вывод первого элемента**  

   Программа создает пременную, обозначенную как `x`, и присваивает ей начальное значение, равное `a0 + 3`. Это будет первый член последовательности, который сразу выводится.

4. **Генерация остальных n элементов**

   Начинаем цикл `for`. Для каждого последующего элемента (от индекса `i=1` до `i=n` включительно).
   На каждой итерации:
    если `i` нечётное (`i%2 == 1`), то к текуещее значение `x` целочисленно делится на 4 ('x = x/4');
    если `i` чётное (`i%2 == 0`), то к текущему значению x прибавляется 3 ( `x = x+3`).
   
5. **Последовательный вывод**

   Каждый вычисленный элемент сразу выводится на экран с пробелом после него. 

Задача 2.

1. **Ввод данных**

   Программа считывает целое положительное число `n` — длину последовательности.
Затем считывает `n` целых чисел и сохраняет их в массив `arr`.

2. **Подготовка счетчика положительных частей**

Инициализируется переменная `positive` со значением `0`. Она будет использоваться для подсчёта количества частей, произведение элементов которых положительно.

3. **Перебор всех возможных подотрезков**

Внешние вложенные циклы (`i` от `0` до `n−1`, `j` от `i` до `n−1`) перебирают все возможные непрерывные подотрезки массива `arr`.

4. **Проверка наличия разделителя (кратного 3) в подотрезке**

Для каждого подотрезка от `i` до `j` проверяется, содержится ли в нём хотя бы одно число, кратное 3:
если такое число найдено, подотрезок помечается как недопустимый (`three = true`) и пропускается;
если кратных 3 нет, подотрезок считается частью согласно условию задачи.

5. **Определение знака произведения части**

Для каждой корректной части (без чисел, кратных 3):
подсчитывается количество отрицательных элементов;
если это количество чётное (и часть не пустая), произведение положительно — счётчик `positive` увеличивается на 1.

6. **Вывод результата**

После анализа всех частей программа выводит значение переменной `positive` — количество частей с положительным знаком произведения.

Задача 3.

1. **Ввод данных**
   
Программа считывает одно целое неотрицательное число `n` типа `long` (для поддержки больших значений).

2. **Обработка особого случая — ноль**
 
Если введённое число равно `0`, программа немедленно выводит `"YES"` (так как `0` считается палиндромом) и завершает работу.

3. **Сохранение исходного значения и подготовка к развороту**
   
Исходное значение числа сохраняется в переменную `original`.
Переменная `reversed` инициализируется нулём — в неё будет записываться число, составленное из цифр исходного в обратном порядке.

4. **Построение перевёрнутого числа**
   
Пока `n` больше нуля:

- последняя цифра `n` (полученная как `n % 10`) добавляется в конец числа `reversed` (`reversed = reversed * 10 + n % 10)`;
  
- число `n` целочисленно делится на 10, отбрасывая последнюю цифру.
  
5. **Сравнение и вывод результата**
   
После завершения разворота программа сравнивает исходное число (`original`) с перевёрнутым (`reversed`):

- если они равны, выводится `"YES"` — число является палиндромом;
  
- иначе выводится `"NO"`.

Задача 4.

1. **Ввод данных**
   
Программа считывает целое положительное число `n` — количество элементов в массиве.
Затем считывает `n` целых чисел и сохраняет их в массив `a`.

2. **Подготовка вспомогательных структур**

Создаётся булев массив `good` длины `n`, в котором будет отмечаться, удовлетворяет ли элемент на соответствующей позиции условию задачи.
Также объявляется флаг `found`, изначально равный `false`, чтобы отслеживать, 
найдены ли вообще подходящие элементы.

3. **Проверка каждого элемента массива**

Для каждого элемента `a[i]` (от `i = 0` до `n−1`):

- подсчитывается, сколько раз он встречается во всём массиве (`count`);
  
- вычисляется сумма цифр его модуля (`sum`):
  
- если элемент равен нулю, сумма цифр принимается равной `0`;
  
иначе число берётся по модулю, и в цикле извлекаются десятичные цифры через остаток от деления на 10.

4. **Сравнение частоты и суммы цифр**
   
Если количество вхождений элемента `count` равно сумме его цифр `sum`, то:

- в массиве `good` на позиции `i` устанавливается значение `true`;
  
- флаг `found` устанавливается в `true`.
  
5. **Формирование результата**
Если ни один элемент не удовлетворил условию (`found == false`), программа выводит строку `"NO"`.
В противном случае:

- перебираются все позиции массива, и для каждой `i`, где `good[i] == true`, значение `a[i]` выводится с пробелом после него;
 
- после вывода всех подходящих элементов добавляется перевод строки.
  
  ### 4. Программа

 Задача 1.
 
```java
import java.io.PrintStream;
import java.util.Scanner;
import java.io.IOException;

public class Main {
    public static Scanner in = new Scanner(System.in);
    public static PrintStream out = System.out;

    public static void main(String[] args) {
        int a0 = in.nextInt();
        int n = in.nextInt();
        int x = a0 + 3;
        out.print(x + " ");
        for (int i = 1; i <= n; i++) {
            if (i % 2 == 1) {
                x /= 4;
                out.print(x + " ");
            } else {
                x += 3;
                out.print(x + " ");
            }
        }
    }
}
