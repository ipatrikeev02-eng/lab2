### 1. Постановка задачи
Задача 1
> Ваша задача – посмотреть, какие значения принимает последовательность при разных a0 и n, вывести закономерность, по которой строится последовательность и запрограммировать ее самостоятельно.
>Ограничение сверху на n – искусственное, чтобы не упал весь сайт (а вы можете выбрать любое разумное ограничение самостоятельно). Если на ваш запрос сервер пятисотит (HTTP 500), значит ваша последовательность быстро растет, и питон сломался обрабатывать ее элементы, попробуйте вывести еще меньше элементов.

Для решения данной задачи необходимо проанализировать последовательность при разных a0 и n и вывести закономерность.

Задача 2.
>Последовательность разбивается на части числами, кратными 3: такие числа выступают разделителями и в часть не входят. Для каждой полученной части нужно определить знак произведения ее элементов. Подсчитайте, сколько частей имеют положительный знак произведения, и выведите это число. Замечание: операцию умножения чисел использовать нельзя.
Для решения данной задачи необходимо разбить последовательность на подпоследовательности, используя числа, кратные 3, как разделители (они не входят в подпоследовательности). Для каждой полученной части требуется определить знак произведения её элементов без использования операции умножения. После этого нужно подсчитать, сколько частей имеют положительное произведение, и вывести это количество.
>Задача 3.
>Определите, является ли десятичная запись числа палиндромом (одинаково читается слева направо и справа налево).
Для решения данной задачи необходимо определить, является ли десятичная запись заданного целого числа палиндромом — то есть совпадает ли она при чтении слева направо и справа налево. Требуется вывести «YES», если число — палиндром, и «NO» в противном случае.
Задача 4.
>Дан массив из n чисел. Вывести все элементы, количество появлений которых равно сумме цифр самого элемента. Если таких элементов нет, вывести "NO".
Для решения данной задачи необходимо проанализировать массив целых чисел и найти все его элементы, у которых количество появлений в массиве в точности равно сумме цифр самого элемента (сумма цифр считается по модулю, без учёта знака). Если таких элементов нет, следует вывести «NO»; иначе — вывести все подходящие элементы (каждый один раз, порядок не важен).
